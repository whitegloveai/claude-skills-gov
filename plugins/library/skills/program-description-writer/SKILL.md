# Program Description Writer

> Drafts a calendar-ready program description and a longer newsletter or press release version. Built for public library programming staff publishing under deadline across multiple channels.

## When to Use

- "Write the calendar copy for next month's STEM Saturday series."
- "Draft a social caption and a newsletter blurb for the resume workshop on the 14th."
- "We need a press release version of the author talk for the city newsletter."
- "Convert these program notes into copy I can paste into the events platform."
- "The branch manager handed me a one-line program idea — turn it into publishable copy."

## Inputs Needed

Required:

- Program name and type (storytime, book club, STEM workshop, job-skills class, civic literacy, author event, craft, movie, etc.)
- Audience (age band — e.g., ages 3 to 5, ages 8 to 12, teens, adults, all ages — or "general")
- Dates and times
- Location (branch name, room, or community center partner site)
- Capacity, if limited
- Registration mechanism (drop-in, registration required, waitlist)

Optional but improves output:

- Prerequisites or materials attendees must bring
- Partner organizations to credit
- Accessibility considerations beyond the standard ADA statement (sensory-friendly hour, ASL interpreter on request, large-print materials)
- Language access — programs offered bilingually or with translation support
- A staff contact for questions

If the audience age band, registration mechanism, or location is missing, ask before drafting. These three are the load-bearing details for citizens.

## Process

1. Confirm the four load-bearing facts: who, what, when, where. Add the fifth — how to attend — by confirming the registration mechanism.
2. Draft the short calendar and social version, target 80 to 150 words. Lead with a hook of one or two sentences. State what attendees will do, who the program is for, registration mechanics, and the accessibility note. Plain language at roughly an 8th-grade reading level.
3. Draft the longer newsletter or press release version, target approximately 300 words. Expand the hook, add context on the program series or partner, include a quote-ready line that staff can attribute to the library director or programming librarian, and close with practical attendance information.
4. Include the standard ADA Title II accessibility statement. If the program is offered bilingually or with translation, add the language access line.
5. If the program touches on a contested topic (banned books week, voter registration drive, immigration legal clinic), apply the same intellectual freedom posture the library uses for collection development — describe the program neutrally, name the partner, and avoid political endorsement. The library serves all residents.
6. Flag any program that would benefit from a parental consent or photo release form so staff can prepare those alongside publication.

## Output Format

Plain markdown with two clearly labeled blocks: SHORT VERSION (calendar and social) and LONG VERSION (newsletter and press release). No emoji. No exclamation points except where genuinely welcoming.

```
PROGRAM: [Program name]
AUDIENCE: [Age band or general]
DATE AND TIME: [Date(s) and time]
LOCATION: [Branch and room]
REGISTRATION: [Drop-in / Register at URL / Capacity]

SHORT VERSION (~80–150 words)
[Hook of 1–2 sentences. What attendees will do.
Who it's for. Registration mechanics.
Accessibility note.]

LONG VERSION (~300 words)
[Expanded hook.
Program description and what attendees will experience.
Partner credit and series context.
Quote-ready line for staff attribution.
Practical attendance information and contact.
Accessibility statement.]

ACCESSIBILITY STATEMENT
The [Library Name] is committed to providing access for all
residents. To request an accommodation, please contact
[staff contact] at least [N] business days before the program.

OPTIONAL LANGUAGE ACCESS LINE
Programs in [language] available on request.
```

Length: typically one page total. Tone: warm and welcoming for citizens, no bureaucratic distance.

## TRAIGA & Compliance Notes

**AI use disclosure — transparency best practice (not a § 552.051 trigger):** Tex. Bus. & Comm. Code § 552.051(b)–(d) requires disclosure when a citizen *interacts directly with an AI system* (e.g., a chatbot, automated voice assistant, or AI-driven service interface). The disclosure trigger is *interaction*, not *output*. This skill produces a static document drafted with AI assistance; it does not put a citizen in direct interaction with AI, so § 552.051 is not engaged.

However, transparency about AI assistance in document drafting is sound practice and should align with the city's adopted AI use policy and the GovAI Coalition's guidance on documenting AI tool use. Consider a brief notation on documents distributed publicly or to the governing body: *"AI-assisted draft, reviewed by [staff title and name]."* This is a policy choice, not a statutory mandate.

**ADA Title II accessibility:** Public library programs are services of a city or county and must be accessible to people with disabilities. The accessibility statement and accommodation request mechanism are not optional. Allow reasonable lead time — commonly five business days — for sign language interpretation or other accommodations.

**Language access:** Where the library serves a substantial non-English-speaking population, programs offered with translation or in additional languages should say so plainly. This is good practice and supports compliance with city language access policies adopted under federal civil rights guidance.

**Intellectual freedom:** Program content selection is governed by the library's collection development and programming policies. Programs that touch on contested topics should be presented neutrally, without political endorsement, consistent with the American Library Association Library Bill of Rights.

**Photo and media release:** If the program will be photographed or recorded for promotional use, the description should note that and direct attendees to the release form. Programs for minors require parental consent.

**Public records:** Program descriptions, attendance counts (in aggregate), and final published copy are public records under Tex. Gov't Code Ch. 552 and are typically retained per the Texas State Library and Archives Commission (TSLAC) Local Government Retention Schedule.
