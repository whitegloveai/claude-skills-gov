# it

Claude skills for local government IT. Built for CIOs, IT directors, security officers, AI governance leads, and the small-shop IT generalist running everything in a 10,000-resident city.

## Skills in this plugin

| Skill | Purpose |
|---|---|
| [it-policy-drafter](skills/it-policy-drafter) | Drafts an IT or acceptable-use policy aligned with NIST controls and state requirements |
| [security-incident-classifier](skills/security-incident-classifier) | Classifies a security event by severity, category, and notification trigger |
| [vendor-risk-questionnaire](skills/vendor-risk-questionnaire) | Drafts or fills a vendor risk questionnaire (CAIQ-style or custom) |
| [ai-governance-checklist](skills/ai-governance-checklist) | Drafts an AI use-case intake form aligned with NIST AI RMF and TRAIGA, modeled on GovAI Coalition templates |
| [system-of-record-mapper](skills/system-of-record-mapper) | Documents a system of record: data classification, authority, retention, integrations |

## Install

```
/plugin install it@whitegloveai-gov
```

## Compliance

These skills are aligned with NIST AI RMF, NIST Cybersecurity Framework, TRAIGA (Tex. Bus. & Comm. Code Ch. 552), HB 3512 (AI literacy training), and Texas DIR cybersecurity reporting rules. Drafts produced here are intended as starting points for the official record; final approval should come from the CIO or governing body.

## Notes on Examples

Examples in this plugin reference the fictional City of Cedar Ridge, TX for continuity across skills. The skill logic is jurisdiction-agnostic — adapt example specifics to your city.
